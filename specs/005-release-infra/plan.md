# Implementation Plan: Release Infrastructure (v0.2.0 cut)

**Branch**: `005-release-infra` | **Date**: 2026-04-18 | **Spec**: [spec.md](./spec.md)
**Input**: [spec.md](./spec.md) — 7 FRs, 5 SCs, 2 user stories.

## Summary

Wire [`cargo-dist`](https://github.com/axodotdev/cargo-dist) v0.31.x as the single release orchestrator. `cargo-dist init` generates `[workspace.metadata.dist]` in `Cargo.toml` + `.github/workflows/release.yml`. On top of the generated workflow, add three guard jobs (tag-version match, CHANGELOG section present, tag reachable from `main`) so a bad tag fails fast before any artifact is built. Write `docs/release.md` as the human procedure companion. Bump version 0.2.0-dev → 0.2.0 and move CHANGELOG `[Unreleased]` content into `[0.2.0] - 2026-04-18` as the same commit.

## Technical Context

**Tooling**: `cargo-dist` v0.31.x (pinned via `cargo install cargo-dist@0.31`), GitHub Actions runners (no self-hosted), `dtolnay/rust-toolchain@stable`.
**Storage**: None. Release artifacts live on GitHub Releases; no SpecERE-side state added.
**Testing**: A throwaway `v0.2.0-testrelease` tag on a disposable branch exercises the workflow end-to-end before the real `v0.2.0` tag. Local reproduction via `dist plan` / `dist build --target <triple>`.
**Targets** (per FR-RI-002): `x86_64-unknown-linux-gnu`, `aarch64-unknown-linux-gnu`, `x86_64-apple-darwin`, `aarch64-apple-darwin`, `x86_64-pc-windows-msvc`.
**Constraints**: Tag push on `main` only (FR-RI-005). No crates.io publish yet (deferred).

## Priors

1. **Pin cargo-dist at 0.31.x.** Compatible with cargo 1.78 toolchain pin. Newer cargo-dist versions have had breaking config changes; explicit pin prevents drift.
2. **Install script format**: shell + powershell for the two Unix/Windows install paths. No msi, no homebrew tap (deferred). This is the `cargo-dist init --installers sh,powershell` default.
3. **Guard jobs over cargo-dist's own validation.** cargo-dist does version-match by default but doesn't enforce CHANGELOG-section-present or reachable-from-main. The three guard jobs go in their own top-level job that precedes the cargo-dist-generated jobs via `needs:`.
4. **CHANGELOG source-of-truth for release notes.** cargo-dist can auto-generate release notes; we override to use `CHANGELOG.md ## [<version>]` section so the human-authored text wins.
5. **No crates.io publish.** Out of scope. Follow-up spec `release-infra-crates-io` will add it.
6. **Tag naming**: strict `v<semver>` — `v0.2.0`, `v0.3.0-rc.1`. Tags without `v` prefix are ignored by the workflow.
